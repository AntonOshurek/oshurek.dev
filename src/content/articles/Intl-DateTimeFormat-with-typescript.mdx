<ContentPart>
	<ArticleTitleL headingTag="h2">Как исправить ошибку с ...</ArticleTitleL>
	<ArticleParagraphM>
		Данная ошибка появилась из за довольно простой причины. Когда мы используем объект с
		литеральными типами, мы должны точно указать какие типы там будут. Для этого, нам нужно пойти в
		глубь intl и достать от туда объект с опциями, и приписать этот тип к нашему объекту. Ниже, я
		предоставляю работающий пример кода.
	</ArticleParagraphM>

```typescript
const fullOptions: Intl.DateTimeFormatOptions = {
	year: 'numeric',
	month: 'short',
	day: '2-digit',
	hour: '2-digit',
	minute: '2-digit',
	timeZone: 'CET',
	hour12: false,
};

formattedDate = new Intl.DateTimeFormat('en-GB', fullOptions).format(d);
```

</ContentPart>

	<ArticleParagraphM>
		В typescript, если мы создадим переменную с текстом, то тип этой переменной не будет string а
		только сам текст из переменной.
	</ArticleParagraphM>
	<ArticleParagraphM>
		Так же, если мы используем литеральный тип, то переменная не будет являться типом string а
		только одним из литералов.
	</ArticleParagraphM>
	<ArticleParagraphM>
		Таким образом, нам нужно внимательнее следить за нашими типами и помнить, что не всё строка что
		кажется строкой)))
